
# Конфигурационные файлы проекта — краткий разбор

## main.css — подключение Tailwind v4

```css
@import "tailwindcss";
```

**Всё.** В Tailwind v4 конфигурация переехала из `tailwind.config.js` прямо в CSS. Больше не нужен отдельный конфиг-файл — кастомные цвета, шрифты, брейкпоинты задаются через `@theme` прямо здесь (если понадобится).

**Почему это важно:** Vite плагин `@tailwindcss/vite` читает этот файл и автоматически генерирует утилитарные классы. Без этого импорта Tailwind не работает.

## index.html — точка входа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Vite App</title>
</head>
<body>
  <div id="app"></div>
  <script type="module" src="/src/main.js"></script>
</body>
</html>
```

**Ключевые моменты:**
- `<div id="app">` — сюда Vue монтирует всё приложение
- `<script type="module" src="/src/main.js">` — Vite загружает `main.js` как ES-модуль
- `lang="ru"` — важно для SEO и скринридеров

**Что стоит добавить:** мета-теги для SEO (description, Open Graph), favicon.

## vite.config.js — конфигурация сборщика

```javascript
export default defineConfig({
  plugins: [
    vue(),
    tailwindcss(),
    vueDevTools(),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
      'swiper/vue': path.resolve(__dirname, 'node_modules/swiper/swiper-vue.mjs'),
      'swiper/modules': path.resolve(__dirname, 'node_modules/swiper/modules/index.mjs'),
      'swiper/css/bundle': path.resolve(__dirname, 'node_modules/swiper/swiper-bundle.css'),
    },
  },
  server: {
    proxy: {
      '/sanatorium': {
        target: 'https://sites.creatrix-digital.ru',
        changeOrigin: true,
        secure: false,
      },
    },
  },
})
```

### Плагины

| Плагин | Зачем |
|--------|-------|
| `vue()` | Компиляция `.vue` файлов (SFC) |
| `tailwindcss()` | Интеграция Tailwind v4 с Vite |
| `vueDevTools()` | Отладка Vue в браузере (Vue DevTools) |

### Alias для Swiper

**Почему это нужно?** Swiper 12 изменил структуру импортов. Без этих alias пришлось бы писать:
```javascript
import { Swiper } from 'swiper/vue/dist/swiper-vue.mjs'  // ❌ длинно
```

С alias:
```javascript
import { Swiper } from 'swiper/vue'  // ✅ коротко
```

### Proxy для API

```javascript
proxy: {
  '/sanatorium': {
    target: 'https://sites.creatrix-digital.ru',
    changeOrigin: true,
    secure: false,
  },
}
```

**Как работает:**
1. Фронт делает запрос: `GET /sanatorium/api/v1/homes`
2. Vite видит префикс `/sanatorium` и перенаправляет на `https://sites.creatrix-digital.ru/sanatorium/api/v1/homes`
3. Браузер думает, что запрос ушёл на `localhost:5173` → **нет CORS-ошибки**
4. Сервер получает запрос, отвечает JSON

**`changeOrigin: true`** — меняет `Host` заголовок на целевой домен (некоторые серверы проверяют)
**`secure: false`** — разрешает самоподписанные SSL-сертификаты (для разработки)

## Связь файлов

```
index.html
    ↓ загружает
main.js (не прислан, но там createApp + router)
    ↓ импортирует
App.vue
    ↓ использует
router.js → монтирует компоненты в <router-view>
    ↓ компоненты импортируют
API сервисы (homes.js, services.js...)
    ↓ используют
client.js (axios с baseURL)
    ↓ Vite proxy перенаправляет
vite.config.js → https://sites.creatrix-digital.ru
    ↓ стилизует
main.css → Tailwind v4
    ↓ генерирует классы
package.json → tailwindcss + @tailwindcss/vite
```

## Ключевые концепции

1. **Tailwind v4 CSS-first** — конфигурация через `@import` и `@theme`, не через JS-файл
2. **Vite alias** — упрощают импорты, особенно для библиотек со сложной структурой (Swiper)
3. **Dev proxy** — решает CORS без настройки бэкенда
4. **Модульная архитектура** — каждый файл отвечает за одну задачу (плагины, алиасы, прокси)
