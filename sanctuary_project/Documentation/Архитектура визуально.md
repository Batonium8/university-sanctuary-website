## Пошаговый сценарий: Пользователь бронирует номер

### Шаг 1: Пользователь кликает "Подробнее" на карточке номера

**Что происходит:**
```javascript
// Rooms.vue
const handleDetailsClick = (roomId) => {
  router.push({ name: 'HomeDetail', params: { id: roomId } });
};
```

**Результат:**
- URL меняется с `/` на `/rooms/1`
- Vue Router находит маршрут `HomeDetail` в `router.js`
- В `<router-view>` монтируется компонент `HomeDetailsView.vue`

---

### Шаг 2: Монтируется HomeDetailsView (обёртка)

**Что происходит:**
```vue
<!-- HomeDetailsView.vue -->
<template>
  <HomeDetail />           ← монтируется компонент деталей
  <div id="bookings">
    <BookingForm />        ← монтируется форма бронирования
  </div>
</template>
```

**Результат:**
- `HomeDetail` начинает загружать данные о номере
- `BookingForm` просто рисуется, ждет действий пользователя

---

### Шаг 3: HomeDetail делает запрос к API

**Что происходит:**
```javascript
// HomeDetail.vue
const loadHome = async (id) => {
  const response = await getHomeById(id);  // ← вызов сервисного слоя
  home.value = response.data.data;          // ← двойная обёртка
  setSelectedHomeId(home.value.id);         // ← сохраняем ID глобально
};

onMounted(() => {
  loadHome(route.params.id);  // id = 1 из URL
});
```

**Поток запроса:**
```
HomeDetail.vue
    ↓ вызывает
getHomeById(1)  [homes.js]
    ↓ использует
apiClient.get('/homes/1')  [client.js]
    ↓ склеивает с baseURL
'/sanatorium/api/v1/homes/1'
    ↓ Vite proxy перенаправляет
https://sites.creatrix-digital.ru/sanatorium/api/v1/homes/1
    ↓ Laravel возвращает
{ data: { id: 1, name: "Стандартный номер", ... } }
    ↓ обратно в компонент
home.value = response.data.data
```

**Результат:**
- `home.value` содержит объект номера
- `setSelectedHomeId(1)` сохраняет ID в глобальное состояние

---

### Шаг 4: Глобальное состояние обновляется

**Что происходит:**
```javascript
// useBooking.js (composable)
export const selectedHomeId = ref(null);

export function setSelectedHomeId(id) {
  selectedHomeId.value = id ? Number(id) : null;
}
```

**Результат:**
- Все компоненты, которые импортировали `selectedHomeId`, мгновенно узнают об изменении
- `BookingForm.vue` теперь "знает", что выбран номер с ID=1

---

### Шаг 5: Пользователь заполняет форму и нажимает "Забронировать"

**Что происходит:**
```javascript
// BookingForm.vue
const handleSubmit = async () => {
  // 1. Валидация
  if (!validateAll()) return;

  // 2. Формирование payload
  const payload = {
    fio: form.fullName,
    phone: form.phone,
    email: form.email,
    check_in_date: form.checkIn,
    check_out_date: form.checkOut,
    count_people: Number(form.guests),
    home_id: Number(form.roomType),  // ← ID номера из dropdown
    comment: form.wishes || null,
  };

  // 3. Отправка на сервер
  try {
    await createReservation(payload);
    successMessage.value = 'Заявка отправлена!';
  } catch (err) {
    // Обработка ошибок...
  }
};
```

**Поток запроса:**
```
BookingForm.vue
    ↓ вызывает

    ↓ использует
apiClient.post('/reservations', payload)  [client.js]
    ↓ склеивает с baseURL
POST '/sanatorium/api/v1/reservations'
    ↓ тело запроса (JSON)
{
  "fio": "Иванов Иван",
  "phone": "+79001234567",
  "email": "test@mail.ru",
  "check_in_date": "2026-06-10",
  "check_out_date": "2026-06-15",
  "count_people": 2,
  "home_id": 3,
  "comment": "Хочу вид на лес"
}
    ↓ Laravel сохраняет в базу
    ↓ возвращает 201 Created
    ↓ компонент показывает успех
```

### Шаг 6: Обработка ошибок (если что-то пошло не так)

**Сценарий: Пользователь не заполнил обязательное поле**

```javascript
catch (err) {
  if (err.response?.status === 422) {
    // Laravel вернул ошибки валидации
    const apiErrors = err.response.data.errors;
    // { "fio": ["ФИО обязательно"], "email": ["Email некорректный"] }
    
    const fieldMapping = {
      fio: 'fullName',
      email: 'email',
      // ... маппинг полей API на поля формы
    };

    // Проставляем ошибки под нужными полями
    Object.keys(apiErrors).forEach(apiField => {
      const formField = fieldMapping[apiField];
      errors[formField] = apiErrors[apiField][0];
    });
  }
}
```

**Результат:**
- Под полем "ФИО" появляется красная надпись "ФИО обязательно"
- Рамка поля становится красной (`:class="{ 'border-red-400': errors.fullName }"`)

## Ключевые паттерны и принципы

### 1. Разделение ответственности (Separation of Concerns)

```
View-обёртки (HomeDetailsView, DoctorView...)
    ↓ отвечают за композицию страницы
Компоненты (HomeDetail, BookingForm...)
    ↓ отвечают за UI и логику
Сервисный слой (homes.js, reservations.js...)
    ↓ отвечают за общение с API
```

**Преимущество:** каждый слой можно менять независимо. Если бэкенд переедет на другой домен, меняем только `client.js`.

---

### 2. Однонаправленный поток данных

```
Родитель → Пропсы → Ребёнок
Ребёнок → Emits → Родитель

Пример:
Rooms.vue → :title, :price → RoomCard.vue
RoomCard.vue → @details-click → Rooms.vue
```

**Преимущество:** данные текут в одну сторону, легко отслеживать, где что изменилось.

---

### 3. Composable как глобальное состояние

```javascript
// useBooking.js
export const selectedHomeId = ref(null);  // один объект в памяти

// HomeDetail.vue
setSelectedHomeId(1);  // меняет значение

// BookingForm.vue
home_id: selectedHomeId.value  // читает то же значение
```

**Преимущество:** передача данных между несвязанными компонентами без пропсов и без тяжёлых библиотек (Pinia/Vuex).

---

### 4. Двойная обёртка API (axios + Laravel)

```javascript
response.data       // ← обёртка axios (тело HTTP-ответа)
response.data.data  // ← обёртка Laravel API Resource

// Пример:
{
  data: {                    // ← axios
    data: {                  // ← Laravel
      id: 1,
      name: "Стандартный номер"
    }
  }
}
```

**Почему так?** Laravel использует формат JSON:API, который стандартизирует структуру ответов.

---

### 5. Реактивность и жизненный цикл

```javascript
onMounted(() => {
  loadRooms();  // запрос при первом рендере
});

watch(() => route.params.id, (newId) => {
  loadHome(newId);  // перезагрузка при изменении URL
});

onUnmounted(() => {
  window.removeEventListener('resize', handleResize);  // очистка
});
```

**Преимущество:** данные загружаются только когда нужны, слушатели событий не висят в памяти после ухода со страницы.

```
       │                                        │
       ▼                                        ▼

```

## Что мы получили

1. **SPA без перезагрузок** — Vue Router меняет компоненты внутри `<router-view>`
2. **Реальное API** — данные загружаются с Laravel-бэкенда, не захардкожены
3. **Глобальное состояние без боли** — composable `useBooking` передает ID номера между компонентами
4. **Обработка ошибок** — валидация на фронте + обработка 422 от сервера
5. **Адаптивность** — Tailwind CSS, мобильные меню, свайпы
6. **Чистая архитектура** — разделение на слои, переиспользуемые компоненты
