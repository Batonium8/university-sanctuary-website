# Компоненты HeroSection, FAQ, About, Footer — разбор вёрстки

## 🎯 HeroSection.vue — первый экран сайта

### Назначение

Это **Hero-блок** — самая первая секция, которую видит пользователь. Её задача: за 3 секунды донести, куда он попал, и вызвать желание остаться.

### Разбор вёрстки

```vue
<section class="relative min-h-dvh flex items-center justify-center overflow-hidden">
```

| Класс | Что делает |
|-------|------------|
| `relative` | Родитель для абсолютно позиционированных детей (картинка, оверлей) |
| `min-h-dvh` | **Dynamic viewport height** — высота = 100% видимой области браузера. В отличие от `vh`, учитывает плавающую адресную строку на мобильных |
| `flex items-center justify-center` | Центрирует содержимое по вертикали и горизонтали |
| `overflow-hidden` | Обрезает всё, что вылезает за границы (важно для картинки) |

### Три слоя (Z-index пирамида)

```vue
<img class="absolute inset-0 z-0 ..." />           <!-- Слой 0: фон -->
<div class="absolute inset-0 z-10 bg-[#EFE6D7BA]" /> <!-- Слой 1: оверлей -->
<div class="relative z-20 ...">                     <!-- Слой 2: текст -->
```

**Почему именно так:**
- `absolute inset-0` = `top:0; right:0; bottom:0; left:0` — элемент растягивается на весь родитель
- `z-0`, `z-10`, `z-20` — чёткая иерархия слоёв
- Оверлей `bg-[#EFE6D7BA]` — полупрозрачный бежевый (BA в hex = ~73% непрозрачности). Делает текст читаемым на любой картинке

### Оптимизация загрузки картинки

```vue
<img
  :src="hero.bgImage"
  loading="eager"
  fetchpriority="high"
  decoding="async"
/>
```

| Атрибут | Зачем |
|---------|-------|
| `loading="eager"` | Грузить **немедленно**, не ждать скролла. Hero-картинка — LCP-элемент (Largest Contentful Paint), от неё зависит скорость загрузки в глазах Google |
| `fetchpriority="high"` | Приоритет запроса выше обычного. Браузер скачает картинку раньше JS/CSS |
| `decoding="async"` | Декодировать картинку в фоновом потоке, не блокируя основной |

**Правило:** `loading="eager"` — только для Hero-картинки. Все остальные картинки на сайте — `loading="lazy"`.

### Типографика

```vue
<p class="font-['Tenor_Sans'] text-[18px] lg:text-[25px]">
<h1 class="font-['Montserrat'] text-[40px] md:text-[80px] lg:text-[128px] leading-[0.9]">
```

- `leading-[0.9]` — межстрочный интервал 90% от размера шрифта. Для огромных заголовков стандартный `leading` даёт слишком много воздуха между строками
- `tracking-tight` — буквы чуть плотнее друг к другу (для больших заголовков выглядит лучше)
- `whitespace-nowrap` — запрет переноса. Заголовок "Чистый Воздух" всегда в одну строку

## 🎯 FAQ.vue — аккордеон вопросов

### Назначение

Секция "Часто задаваемые вопросы" с раскрывающимися блоками. Пользователь кликает на вопрос — плавно появляется ответ.

### Разбор вёрстки

```vue
<div class="border-2 border-[#142C12] rounded-lg overflow-hidden"
     :class="{ 'bg-[#142C12]/5': openIndexes.includes(index) }">
```

**Динамический фон:** когда вопрос открыт, блок получает лёгкий тёмный фон (`/5` = 5% непрозрачности). Это визуальная обратная связь — пользователь сразу видит, что активно.

### Кнопка с иконкой "+"

```vue
<button class="w-full flex items-center justify-between px-5 md:px-8 py-4 md:py-5 text-left">
  <span>{{ item.question }}</span>
  <span :class="{ 'rotate-45': openIndexes.includes(index) }">+</span>
</button>
```

**Приём с поворотом:** вместо замены иконки "+" на "×" мы просто поворачиваем плюс на 45°. Получается крестик. Это **элегантное решение** — одна SVG/символ, два состояния через CSS-трансформацию.

### Анимация раскрытия через max-height

```vue
<div class="overflow-hidden transition-all duration-300 ease-in-out"
     :style="{
       maxHeight: openIndexes.includes(index) ? '500px' : '0px',
       opacity: openIndexes.includes(index) ? 1 : 0
     }">
```

**Почему `max-height`, а не `height`?**
- `height: auto` нельзя анимировать в CSS (браузер не знает, куда стремиться)
- `max-height: 500px` — задаём "потолок", который заведомо больше контента. Анимация идёт от 0 до 500px
- `overflow-hidden` — обрезает всё, что не влезло в текущий max-height

**Двойная анимация:**
- `maxHeight` — отвечает за высоту
- `opacity` — отвечает за плавное появление текста (иначе текст "выстреливает" резковато)

### Хранение состояния

```javascript
const openIndexes = ref([]);

const toggleQuestion = (index) => {
  const currentIndex = openIndexes.value.indexOf(index);
  if (currentIndex === -1) {
    openIndexes.value.push(index);  // Открыть
  } else {
    openIndexes.value.splice(currentIndex, 1);  // Закрыть
  }
};
```

**Массив индексов, а не один boolean** — позволяет открывать **несколько вопросов одновременно**. Если бы хотели "только один открытый" — хранили бы один индекс.

### will-change для производительности

```css
.overflow-hidden {
  will-change: max-height, opacity;
}
```

`will-change` говорит браузеру: "Этот элемент скоро будет анимироваться, подготовь GPU-ускорение заранее". Без этого первая анимация может подтормаживать.

## 🎯 About.vue — секция "О нас" со сложной сеткой

### Назначение

Текст о санатории + галерея из 5 фотографий. На десктопе — двухколоночный макет с текстом слева и сеткой картинок справа. На мобильном — текст сверху, свайп-галерея снизу.

### Разбор вёрстки: сетка 12 колонок

```vue
<div class="grid grid-cols-1 lg:grid-cols-12 gap-10">
  <div class="lg:col-span-7">Текст</div>
  <div class="lg:col-span-5">Картинки</div>
</div>
```

**Почему 12 колонок?** Это стандартная сетка (как в Bootstrap). Позволяет гибко делить пространство: 7+5, 8+4, 6+6 и т.д.

### Десктопная галерея: две колонки по 3 и 2 картинки

```vue
<div class="hidden lg:grid lg:grid-cols-2 gap-3 h-full">
  <div class="flex flex-col gap-3">
    <div v-for="src in images.slice(0, 3)">
      <img />
    </div>
  </div>
  <div class="flex flex-col gap-3">
    <div v-for="src in images.slice(3)">
      <img />
    </div>
  </div>
</div>
```

**Приём:** массив из 5 картинок разрезается на две части (`slice(0, 3)` и `slice(3)`), каждая часть рендерится в свою колонку. Получается асимметричная сетка 3+2 — выглядит интереснее, чем ровная.

### Эффект hover на картинках

```vue
<div class="group border-4 border-[#777C5C]">
  <img class="transform group-hover:scale-105 transition-transform duration-700" />
</div>
```

**Как работает:**
- `group` на родителе → `group-hover:` на ребёнке
- При наведении на **рамку** (родителя) картинка внутри плавно увеличивается на 5%
- `duration-700` — медленная анимация (0.7с), создаёт ощущение "дороговизны"
- `border-4 border-[#777C5C]` — толстая зелёная рамка в стиле бренда

### Мобильная галерея: свайп

```vue
<div class="lg:hidden flex gap-4 overflow-x-auto pb-6 snap-x snap-mandatory">
  <div class="snap-center flex-none w-[85vw] h-72">
    <img />
  </div>
</div>
```

**Нативный свайп без JS:**
- `overflow-x-auto` — горизонтальный скролл
- `snap-x snap-mandatory` — карточки "прилипают" к центру при скролле
- `snap-center` — точка прилипания = центр карточки
- `w-[85vw]` — карточка занимает 85% ширины экрана (видно край следующей — подсказка, что можно листать)
- `flex-none` — карточка не сжимается, сохраняет ширину

### Разные ключи для мобильной и десктопной версий

```vue
:key="'mob-' + index"      <!-- мобильная -->
:key="'desk-l-' + i"       <!-- десктоп левая колонка -->
:key="'desk-r-' + i"       <!-- десктоп правая колонка -->
```

**Зачем?** Если бы ключи были просто `index`, Vue мог бы перепутать элементы при переходе между брейкпоинтами. Префиксы гарантируют уникальность.

## 🎯 Footer.vue — подвал сайта

### Назначение

Двухколоночный макет: слева навигация, справа контакты и соцсети.

### Разбор вёрстки

```vue
<footer class="bg-[#EFE6D7] px-4 sm:px-6 lg:px-8 py-8 md:py-10">
  <div class="max-w-360 mx-auto flex flex-col lg:flex-row lg:justify-between gap-6">
    <nav class="flex flex-col gap-3">Ссылки</nav>
    <div class="flex flex-col items-start lg:items-end gap-4">Контакты</div>
  </div>
</footer>
```

**Адаптивность:**
- Мобильный: `flex-col` — колонки друг под другом
- Десктоп (`lg:`): `flex-row lg:justify-between` — колонки в ряд, разнесены по краям
- `lg:items-end` — на десктопе правая колонка выравнивается по правому краю

### Иконки соцсетей

```vue
<a class="social-icon" aria-label="Telegram">
  <svg viewBox="0 0 41 42">...</svg>
</a>
```

```css
.social-icon {
  width: 42px;
  height: 42px;
  display: flex;
  align-items: center;
  justify-content: center;
  transition: opacity 0.3s ease, transform 0.3s ease;
}

.social-icon:hover {
  opacity: 0.7;
  transform: scale(1.05);
}
```

**Приёмы:**
- Фиксированный размер 42×42px — иконки одинаковые независимо от размера SVG
- `viewBox` у SVG — масштабируется без потери качества
- `aria-label` — для скринридеров (доступность)
- Hover: лёгкое уменьшение прозрачности + увеличение на 5% — "живой" отклик

### Ссылки-якоря

```javascript
const footerLinks = [
  { label: 'О нас', href: '#about' },
  { label: 'Услуги', href: '#services' },
  // ...
];
```

В обновлённой версии добавлен `navigateToSection` — функция, которая ищет элемент по ID и скроллит к нему. Если элемент не найден (мы не на главной) — переход на главную + скролл.

##  Общие CSS-приёмы, которые встречаются во всех компонентах

### 1. `object-cover` для картинок

Заполняет контейнер без искажений, обрезая лишнее. Используется везде, где картинка должна вписаться в блок фиксированного размера.

### 2. `group` + `group-hover:`

Позволяет реагировать на hover родителя, меняя стили ребёнка. Используется в карточках услуг, врачей, галерее About.

### 3. `snap-x snap-mandatory snap-center`

Нативный CSS-свайп без JS. Работает в Safari, Chrome, Firefox на мобильных. Используется в FAQ-мобильной галерее, каруселях.

### 4. `transition-all duration-300`

Плавная анимация всех изменяемых свойств за 300мс. Стандартное время для UI-анимаций — достаточно быстро, чтобы не раздражать, достаточно медленно, чтобы заметить.

### 5. `!pt-32` / `!important` в Tailwind

Восклицательный знак форсирует применение стиля, перебивая другие классы. Нужен для отступа под фиксированный Header на детальных страницах.

### 6. `:deep()` селектор

Пробивает scoped-изоляцию Vue, позволяя стилизовать дочерние компоненты (например, точки пагинации Swiper).

### 7. Динамические классы через `:class`

```vue
:class="{ 'border-red-400': errors.fullName }"
:class="{ 'rotate-45': isOpen }"
```

Объектный синтаксис: ключ — класс, значение — условие. Применяется, если условие `true`.

## 🐛 Типичные баги в вёрстке

1. **Картинка растягивается** → забыл `object-cover`, добавил только `w-full h-full`
2. **Текст наезжает на Header** → забыл `!pt-32` на детальных страницах
3. **Свайп не работает на iOS** → забыл `snap-mandatory` (только `snap-x` недостаточно)
4. **Анимация height не работает** → пытался анимировать `height: auto`. Решение: `max-height` с фиксированным значением
5. **Мобильная галерея ломает десктопную** → забыл `lg:hidden` / `hidden lg:grid`
6. **Z-index не работает** → забыл `relative` у родителя (absolute позиционирование работает только внутри relative/absolute/fixed родителя)
7. **Ссылки в Footer не скроллят** → используешь `<a href="#about">` вместо обработчика с `window.scrollTo`. В SPA якоря не работают автоматически
