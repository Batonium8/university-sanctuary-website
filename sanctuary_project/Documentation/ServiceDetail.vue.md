# Компонент ServiceDetail.vue — страница детальной информации об услуге

##  Назначение

`ServiceDetail.vue` — это отдельная страница, которая показывает **подробную информацию об одной конкретной услуге**. Она загружает данные с сервера по `ID` услуги, который берется из адреса страницы (например, `/services/1`), и красиво их отображает.

---

## 🧱 Верстка (Layout)

### 1. Отступы и фон

```vue
<section class="py-12 md:py-16 px-4 sm:px-6 lg:px-8 bg-[#EFE6D7] !pt-38 min-h-screen">
```

* `bg-[#EFE6D7]` — наш фирменный бежевый фон.
* `min-h-screen` — страница всегда занимает минимум 100% высоты экрана, даже если контента мало. Это нужно, чтобы футер не "подпрыгивал" вверх.
* **`!pt-38`** — очень важный класс. Восклицательный знак `!` в Tailwind означает **принудительное применение**. Он добавляет огромный отступ сверху (152px), чтобы фиксированная шапка (Header) не перекрывала заголовок услуги.

### 2. Паттерн "Три состояния" (State Machine)

В шаблоне используется классический UX-паттерн для страниц, которые грузят данные. Мы показываем **только один** из трех блоков в любой момент времени:

```vue
<div v-if="loading">...</div>
<div v-else-if="error">...</div>
<div v-else-if="service">...</div>
```

1. **`v-if="loading"`**: Показываем анимированный текст "Загрузка...". Пользователь понимает, что сайт не завис, а работает.
2. **`v-else-if="error"`**: Если API вернул ошибку (например, 404 или нет интернета), показываем текст ошибки и **кнопку "Попробовать снова"**. Кнопка просто вызывает ту же функцию загрузки.
3. **`v-else-if="service"`**: Если данные успешно пришли, рендерим саму карточку с фото и описанием.

### 3. Сетка контента (Flexbox)

```vue
<div v-else-if="service" class="flex flex-col lg:flex-row gap-8 lg:gap-12">
  <div class="lg:w-1/2"> <!-- Картинка --> </div>
  <div class="lg:w-1/2 flex flex-col"> <!-- Текст --> </div>
</div>
```

* **Мобильные устройства**: `flex-col` — блоки идут друг под другом (сначала картинка, потом текст).
* **Десктоп (`lg:`)**: `flex-row` — блоки встают в ряд. `lg:w-1/2` делит экран ровно пополам (50% на 50%).
* `gap-8 lg:gap-12` — расстояние между колонками. На десктопе оно больше для "воздуха".

### 4. Работа с изображением

```vue
<div class="rounded-lg overflow-hidden bg-[#D9D0C1]">
  <img :src="service.image" class="w-full h-[400px] md:h-[500px] object-cover" />
</div>
```

* `bg-[#D9D0C1]` — цветной плейсхолдер. Если картинка долго грузится или у нее прозрачный фон, под ней не будет белой дыры.
* `object-cover` — главное правило для картинок. Оно говорит браузеру: "Заполни весь блок, обрезав лишнее по краям, но **не растягивай** и не сплющивай картинку".

## ️ Функциональность и логика

### 1. Получение ID из URL

```javascript
import { useRoute } from 'vue-router';
const route = useRoute();
// route.params.id будет равен "1", если мы на странице /services/1
```

Компонент сам не знает, какую услугу показывать. Он смотрит на адресную строку браузера через `useRoute()` и берет оттуда `id`.

### 2. Функция загрузки `loadService(id)`

```javascript
const loadService = async (id) => {
  if (!id) return; // Защита: если id нет, ничего не делаем

  loading.value = true;   // 1. Включаем спиннер
  error.value = null;     // 2. Сбрасываем старую ошибку
  service.value = null;   // 3. Сбрасываем старые данные (чтобы шаблон скрылся)

  try {
    const response = await getServiceById(id);
    service.value = response.data.data; // Сохраняем данные
  } catch (err) {
    // Обработка ошибок
    if (err.response?.status === 404) {
      error.value = 'Услуга не найдена.';
    } else {
      error.value = 'Не удалось загрузить информацию об услуге.';
    }
  } finally {
    loading.value = false; // 4. Выключаем спиннер (в любом случае: успех или ошибка)
  }
};
```

**Почему так много строчек?**
* **Защита от "мигания"**: Мы сбрасываем `service.value = null`, чтобы при переходе с одной услуги на другую пользователь на долю секунды увидел спиннер, а не старые данные предыдущей услуги.
* **`finally`**: Этот блок выполнится всегда. Если мы забудем выключить `loading.value = false` в `catch`, спиннер будет крутиться вечно. `finally` страхует от этого.

### 3. Магия `watch` (Наблюдатель)

```javascript
onMounted(() => {
  loadService(route.params.id);
});

watch(() => route.params.id, (newId) => {
  loadService(newId);
});
```

**Зачем нужен `watch`, если есть `onMounted`?**
Представь: ты на странице `/services/1`. Внизу есть рекомендации, и ты кликаешь на другую услугу `/services/2`. 
* В обычном сайте страница бы перезагрузилась. 
* В Vue (SPA) компонент `ServiceDetail` **не уничтожается**, он просто обновляется. `onMounted` второй раз **не сработает**.
* Чтобы поймать изменение ID в адресной строке без перезагрузки, мы используем `watch`. Он "следит" за `route.params.id` и как только он меняется — вызывает `loadService(newId)`.

## 🧠 Ключевые концепции

1. **Условный рендеринг состояний** (`v-if / v-else-if`) — лучший способ показать пользователю, что происходит с запросом.
2. **Принудительные стили Tailwind** (`!pt-38`) — используются, чтобы перебить другие классы или гарантировать отступ под фиксированную шапку.
3. **`watch` на параметры роута** — обязательный паттерн для детальных страниц в Vue Router, чтобы навигация между однотипными страницами работала без перезагрузки.
4. **Двойная обертка API** (`response.data.data`) — помним, что первый `data` от axios, второй от Laravel.
